  • Grepathy — 记录并分析 Claude 做出的未经批准的决策，帮助团队复盘 agent 行为与权限边界。[HN 18👍 43💬]
  • Libretto PR agents — 自动修复失败的 Playwright scripts，把浏览器测试调试交给专用 PR agent。[HN 16👍 1💬]
  • Goku — 基于 WASM 与 wllama 的 LLM inference 和 model manager，可在浏览器端管理与运行模型。[HN 9👍 2💬]
  • hallmark — 面向 Claude Code、Cursor 和 Codex 的 anti-AI-slop design skill，今日新增 3372 stars。[GitHub Trending]
  • openinterpreter — 兼容 Codex 的 coding agent，面向 Kimi K3 等 open models，今日新增 661 stars。[GitHub Trending]
  • Graphify — 把代码、SQL schema、脚本、文档、论文、图片和视频转成可查询 knowledge graph，服务 Claude Code、Codex、OpenCode、Cursor 等 coding agents。今日新增 1107 stars。[GitHub Trending]
  • agent-browser — 面向 AI agents 的 browser automation CLI，今日新增 81 stars。[GitHub Trending]

🐦 开发者动态​

  • Josh Woodward（Google Labs、Gemini 与 Google AI Studio VP）：Gemini Spark 扩大 Ultra 用户覆盖，并新增编辑 Google Docs、读取 Sheets/Slides comments、速度提升超过 50%、跨来源 parallel processing 四项能力。
  • Boris Cherny（Claude Code）：在 agent 时代，lint rules、CI、e2e tests 等 automation 的价值更高，因为它们能把一次性修复变成永久能力，并降低新人参与代码库的门槛。
  • Thibault Sottiaux（Codex 与 ChatGPT）：OpenAI 调查了 GPT-5.6 在 full-access、无 sandbox 场景下误删文件的问题，将通过 developer message、权限引导和 harness safeguards 降低风险。
  • Thariq（Claude Code）：理想的 prompting 方式是 thin prompts、thick artifacts + context、thin skills。
  • Guillermo Rauch（Vercel CEO）：Vercel Sandbox 日均创建超过 350 万个，DAU 按月增长 100%，并分享了 Sandbox 与 Web Analytics API 服务 agent 工作流的方向。
  • Aaron Levie（Box CEO）：企业 agent 落地的关键仍是 change management、数据与权限建模；把 engineers 嵌入业务团队能显著加速 workflow 转型。
  • Zara Zhang（Builder）：要让 agent 真正在公司内部工作，组织需要被设计成可读；Shopify 只开放 public channels 的做法还带来了 peer learning。
  • Garry Tan（Y Combinator CEO）：portable skill files 可以减少对单一 frontier model 的依赖。
  • Peter Yang（AI tutorials 与 interviews）：认为 ChatGPT Live 与 Codex 的 tools、plugins、browser 能力仍未充分打通，语音助手应能直接执行文档、邮件、会议和 coding 任务。

🎙️ 播客摘要​

The Founder of a $1.5B AI Company on What Comes After the First Wave of AI Apps（AI & I by Every）邀请 Granola 联合创始人兼 CEO Chris Pedregal，讨论第一波 AI 应用之后的产品机会。对话认为，meeting notes 只是入口，真正的竞争在于谁能拥有 AI-native 工作方式的主界面。Granola 正在尝试主动生成 pre-meeting briefs，在用户需要时提供关于会议、人物与上下文的即时准备。嘉宾也强调，AI startup 即使运行良好仍像 knife fight，产品团队需要同时处理竞争、验证和长期塑形。节目还谈到“bring your own agent”，以及 API 和 MCP 变得更强之后，应用如何从单点功能转向 agent-native 平台。整体观点是，当前仍处于 computing revolution 的早期，应用层的形态和价值边界远未定型。
