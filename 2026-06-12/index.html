🎙️ 播客摘要​

AI & I by Every — How Anthropic Uses Claude Fable 5 With Mike Krieger​

Fable 5 不是一款需要你学会"用更好 prompt"的模型——它是第一款迫使你彻底重新思考"什么是完成工作"的 AI。

Anthropic Labs 负责人、Instagram 联合创始人 Mike Krieger 分享了他几周深度使用 Fable 5 后的真实体验。最大的变化在于时间维度的根本性转变：过去使用 Claude Code 的方式是写 prompt、等结果、检查、继续写下一个 prompt；而现在，他可以晚上给 Claude 布置一个复杂任务，"给它道晚安"，第二天醒来发现它凌晨两点就已经完成并在等下一步指示。遇到远程服务挂了，它会自动写一个占位后端、记录问题、等服务恢复后自行修复——这种自主性和持续性的跨越是前所未有的。

Krieger 的核心工作流变成了"大量架构规划在前 + 长周期执行在后"。他会先用 Fable 做 1-2 小时的设计讨论——生成 HTML 原型、画架构图、写 markdown 文档——然后让多个 Claude 会话并行执行各个模块。他把这比作"同时开 5-6 个标签页处理长期任务"。他还展示了一个周末用 Fable 构建的媒体追踪应用，甚至实现了"从应用内部自修改"的功能：长按聊天按钮即可通过 agent 提交修改请求，预览后即时生效。

一个关键结论是：Fable 5 的"深度思考"能力让它不适合所有场景。Krieger 发现自己在 iOS app 上特意切换到 Sonnet 来问体育比赛比分之类的问题——"用火箭筒打蚊子没必要"。他认为产品层面需要帮用户做更好的模型选择，而不是让人自己判断当前任务需要多大"脑力"。
